# Техническая спецификация проекта `ai_feedback_tg_and_apps_3`

## 1. Назначение проекта

Telegram-бот для отработки навыка дачи обратной связи в диалоговом формате.

Пользователь:
- заходит в бот;
- выбирает игру;
- выбирает сценарий внутри игры;
- ведёт диалог с ИИ-агентом;
- завершает диалог вручную или по таймеру;
- получает последовательную аналитику по завершённому диалогу.

Администратор:
- управляет UI-текстами;
- управляет играми и игровыми промтами;
- управляет аналитическими промтами;
- изменяет пароль админки;
- использует гостевой режим просмотра админки.

## 4. Пользовательский сценарий

### 4.1 Первый запуск
1. Пользователь отправляет `/start`.
2. Система проверяет наличие `user_id` в таблице `users`.
3. Если пользователя нет:
   - создаётся запись в `users`;
   - бот запрашивает имя пользователя.
4. После ввода имени:
   - имя сохраняется в `users`;
   - пользователю показывается стартовый экран.

### 4.2 Повторный запуск
1. Пользователь отправляет `/start`.
2. Если пользователь уже существует и имя заполнено:
   - открывается стартовый экран.

### 4.3 Стартовый экран
На стартовом экране:
- приветственное сообщение;
- техническая информация для отладки;
- кнопки первого уровня:
  - игры из таблицы `games`;
  - `Энциклопедия`;
  - `Личный кабинет`.

### 4.4 Выбор игры
1. Пользователь нажимает кнопку игры.
2. Бот открывает меню игры.
3. Показывается приветствие игры.
4. Показываются сценарии второго уровня из `ui_texts`.

### 4.5 Запуск сценария
1. Пользователь выбирает сценарий второго уровня.
2. Бот:
   - получает `game_alias` из `ui_texts`;
   - находит игровой промт в `game_prompts`;
   - проверяет `is_active`.
3. Если промт активен:
   - создаётся `dialog_id`;
   - запускается FSM игрового диалога;
   - запускается таймер диалога;
   - отправляются `conditions`;
   - отправляется изображение сценария, если оно есть;
   - отправляется приветствие сценария.

### 4.6 Ведение диалога
На каждом пользовательском сообщении:
1. Сообщение сохраняется в `dialog_messages`.
2. Из БД берутся последние 15 пар реплик (30 сообщений).
3. Формируется контекст для ИИ.
4. Пользователю отправляется сообщение `[Думаю...]`.
5. Выполняется запрос в ИИ.
6. Сообщение `[Думаю...]` удаляется.
7. При необходимости снова отправляется изображение сценария.
8. Ответ ИИ сохраняется в `dialog_messages`.
9. Ответ ИИ отправляется пользователю.
10. Под сообщением выводится кнопка `Дай обратную связь`.

### 4.7 Завершение диалога
Диалог завершается двумя способами:
- вручную кнопкой `Дай обратную связь`;
- автоматически по таймеру.

### 4.8 Аналитика завершённого диалога
После завершения диалога:
1. Система получает **весь диалог** по `dialog_id` из `dialog_messages`.
2. Сообщения сортируются по `created_at`, затем по `id`.
3. Собирается единый текстовый блок вида:
   - `Пользователь сказал - ...`
   - `ИИ ответил - ...`
4. Полный текст диалога печатается в консоль как отладочная информация.
5. Из таблицы `analytics_prompts` выбираются все аналитические промты для текущего `game_id`.
6. Для каждого аналитического промта:
   - в ИИ передаётся текст аналитического промта;
   - в ИИ передаётся полный текст диалога;
   - ИИ возвращает JSON:
     - `rating`
     - `text`
   - результат сохраняется в `user_results`.
7. После выполнения всех аналитик:
   - пользователю по очереди отправляются результаты;
   - выдерживается пауза 0.2 секунды между сообщениями;
   - отправляется общая сумма баллов;
   - пользователь переводится на стартовый экран.

## 5. Админка

### 5.1 Вход в админку
Команда: `/admin`

Логика:
1. Бот запрашивает пароль.
2. Если введён пароль `111111`:
   - открывается гостевой режим;
   - разрешён просмотр;
   - изменения не сохраняются.
3. Если введён корректный обычный пароль:
   - дополнительно проверяется `user_id`;
   - в админку допускается только `467116941`.
4. Если пароль верный, но `user_id` не совпадает:
   - создаётся запись в `admin_login_incidents`;
   - пользователь возвращается на `/start`.

### 5.2 Главное меню админки
Кнопки:
- Изменить приветствие главного экрана
- Изменить приветствие админки
- Изменить текст на кнопках
- Изменить пароль админки
- Работа с промтами и играми
- Анализ
- Выйти из админки

### 5.3 Работа с текстами
#### Изменение приветствий
- показывается текущий текст;
- кнопки `Изменить` и `Отмена`;
- после ввода нового текста:
  - если длина меньше 10 символов — возврат в админку;
  - иначе текст обновляется.

#### Изменение текста кнопок
- выводится список всех кнопок из `ui_texts` с `type=button`;
- выбирается кнопка;
- вводится новый текст;
- текст сохраняется.

### 5.4 Смена пароля админки
1. Запрос текущего пароля.
2. Проверка текущего пароля.
3. Запрос нового пароля.
4. Повторный запрос нового пароля.
5. При успехе пароль перезаписывается в `passwords` в виде хэша.

### 5.5 Работа с промтами и играми
Подменю:
- Добавить игру
- Добавить промт
- Изменить промты
- Активировать/деактивировать
- Удалить промт
- Удалить игру
- Назад

#### Добавить игру
1. Ввод названия игры.
2. В таблицу `games` создаётся запись:
   - `name`
   - `game_id`
3. Для игры создаётся приветствие вида `{game_id}_greeting`.

#### Добавить промт
Поддерживается как отдельный административный сценарий.

#### Изменить промты
Поддерживается как отдельный административный сценарий.

#### Активировать/деактивировать
Поддерживается как отдельный административный сценарий.

#### Удалить промт
Поддерживается как отдельный административный сценарий.

#### Удалить игру
1. Показывается список игр.
2. Пользователь выбирает игру.
3. Запрашивается подтверждение удаления.
4. Связанные промты переносятся в `deleted_prompts`.
5. Связанные данные удаляются.

### 5.6 Аналитика
Подменю:
- Новая аналитика
- Изменить аналитику
- Удалить аналитику
- Назад

#### Новая аналитика
1. Выбор игры.
2. Ввод текста аналитического промта.
3. Бот отправляет промт в ИИ.
4. ИИ возвращает:
   - `comment`
   - `alias`
   - `header`
5. Запись создаётся в `analytics_prompts`.
6. Показываются кнопки:
   - `Ещё один промт в эту аналитику`
   - `Назад`

#### Изменить аналитику
1. Показывается список аналитических промтов:
   - `comment | header`
2. Пользователь выбирает запись.
3. Вводится новый текст аналитического промта.
4. Новый текст отправляется в ИИ.
5. ИИ возвращает только:
   - `header`
   - `comment`
6. `alias` сохраняется прежним.
7. Запись обновляется.

#### Удалить аналитику
1. Показывается список аналитических промтов:
   - `comment | header`
2. Пользователь выбирает запись.
3. Показывается подтверждение удаления.
4. При подтверждении:
   - аналитический промт переносится в `deleted_prompts`;
   - текущая запись удаляется из `analytics_prompts`.

## 8. Логические связи таблиц

Прямые SQL foreign key в текущей реализации не обязательны.
Связи поддерживаются логически через поля подстановки.

### Связи по играм
- `games.game_id` → `game_prompts.game_id`
- `games.game_id` → `ui_texts.game`
- `games.game_id` → `dialog_messages.game_id`
- `games.game_id` → `analytics_prompts.game`
- `games.game_id` → `user_results.game_id`

### Связи по игровым сценариям
- `game_prompts.alias` → `ui_texts.game_alias`
- `game_prompts.alias` → `dialog_messages.subgame_id`
- `game_prompts.alias` → `user_results.subgame_id`

### Связи по диалогам
- `dialog_messages.dialog_id` → `user_results.dialog_id`

### Связи по аналитике
- `analytics_prompts.alias` → `user_results.analitics_alias`

### Связи по пользователям
- `users.user_id` → `user_results.user_id`
- Telegram `user_id` → `passwords.user_id`
- Telegram `user_id` → `admin_login_incidents.user_id`

## 9. Работа с ИИ

### 9.1 Игровой ИИ
Используется для ведения диалога.

На вход подаётся:
- системный игровой промт;
- последние 15 пар реплик.

На выходе:
- обычный текст ответа ИИ.

### 9.2 ИИ для метаданных аналитики
Используется в админке.

На вход подаётся:
- текст аналитического промта.

На выходе JSON:
- при создании аналитики:
  - `header`
  - `comment`
  - `alias`
- при редактировании аналитики:
  - `header`
  - `comment`

### 9.3 ИИ для анализа завершённого диалога
Используется после окончания диалога.

На вход подаётся:
- аналитический промт;
- полный текст диалога.

На выходе JSON:
- `rating`
- `text`

- попытки входа в админку;

## 12. Изображения

Для игрового сценария поддерживаются:
- `img_id` — если файл уже известен Telegram;
- `img_path` — если файл хранится локально.
